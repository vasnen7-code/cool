# W Game — Phase 1 (Python / Flask / MySQL)

هذه هي **المرحلة الأولى** من المشروع: الأساس الكامل (قاعدة البيانات، نظام الحسابات،
آلية اللعب الأساسية: الضغط اليدوي + الروبوت + الجمجمة + الجواهر). المراحل القادمة
ستضيف: المتجر، الصناديق، الشخصيات، الحيوانات، العوالم، الزعماء، المهام، الإنجازات،
عجلة الحظ، الأحداث، ولوحة أدمن كاملة — فوق نفس الهيكل بدون أي تعديل جذري.

## التشغيل

```bash
cd wgame
python -m venv venv && source venv/bin/activate   # اختياري لكن موصى به
pip install -r requirements.txt

cp .env.example .env
# عدّل .env ببيانات اتصال MySQL الحقيقية عندك

mysql -u root -p < database/schema.sql
python database/seed_admin.py     # يضبط كلمة مرور الأدمن الحقيقية

python app.py
# افتح المتصفح على http://localhost:5000

# في نافذة طرفية (terminal) ثانية، شغّل بوت OTP بجانب الموقع:
python telegram_bot.py
```

## نظام التحقق عبر OTP (تيليجرام)

- **التسجيل:** يوزر + آيدي تيليجرام + باسورد + تأكيد + رمز OTP (5 أرقام).
  المستخدم يضغط "إرسال الرمز" بالموقع (يولّد الرمز في قاعدة البيانات فقط، لا يُعرض
  بالمتصفح أبدًا)، ثم يفتح بوت **OTP Bot** (`https://t.me/APP1OTPBOT`)، يضغط زر
  **🔑 OTP** بالبوت فيرسله له البوت مباشرة، يرجع للموقع ويكتبه.
- **استعادة كلمة المرور:** نفس الفكرة، لكن الرمز يُرسَل تلقائيًا لآيدي التيليجرام
  المسجَّل مسبقًا مع الحساب (وليس آيدي جديد يُدخله المستخدم) — حماية أقوى.
- **البوت** (`telegram_bot.py`) فيه أيضًا زر **👨‍💻 المطور** (`t.me/Y0YY22`) وزر
  **📢 قناة المطور** (`t.me/sc_fn`)، وهما زرّي رابط ثابتين بدون أي منطق خلفي.
- الرمز صالح 5 دقائق افتراضيًا (`OTP_TTL_SECONDS`)، ولا يمكن طلب رمز جديد قبل مرور
  45 ثانية على الأقل (`OTP_RESEND_COOLDOWN_SECONDS`) لمنع السبام — كلاهما قابل للتعديل من `config.py`.
- ⚠️ **توكن البوت مكتوب حاليًا كقيمة افتراضية في `config.py`.** لا ترفع هذا الملف
  لأي مستودع عام؛ في الإنتاج ضع التوكن في `.env` فقط (المتغير `WGAME_TELEGRAM_BOT_TOKEN`
  موجود بالفعل في `.env.example`) واحذف القيمة الافتراضية من الكود.

## آلية الإنتاج (كما طُلب بالتحديد)

| المصدر | كيف يعمل | مستقل عن |
|---|---|---|
| **الضغط اليدوي** | يعتمد على مستوى "الرأس" + فرصة Critical | كل شيء آخر |
| **الروبوت** | ينتج W تلقائيًا كل ثانية بمعدله الخاص | الجمجمة والضغط و Auto Click |
| **الجمجمة** | تنتج W تلقائيًا كل ثانية بمعدل مختلف عن الروبوت | الروبوت والضغط |
| **Auto Click (من المتجر، مؤقت)** | يحاكي ضغطات تلقائية بمعادلة الضغط اليدوي | مصدر مستقل بحد ذاته |

كل مصدر من الأربعة له **نسبة ظهور جواهر خاصة به** (جدول `gem_drop_rules`)، وتُسجَّل
مساهمة كل مصدر بشكل منفصل في `user_stats` (`total_w_from_click`, `total_w_from_robot`,
`total_w_from_skull`, `total_w_from_auto_click`).

## ملاحظات أمان مطبّقة فعليًا

- كل استعلام SQL في المشروع يمر عبر Prepared Statements (`%s` placeholders)، ولا يوجد
  أي تجميع نصوص (string concatenation) لبناء استعلامات في أي مكان.
- كلمات المرور: `werkzeug.security.generate_password_hash` (pbkdf2:sha256) — مكافئ مباشر لـ `password_hash`/`password_verify`.
- CSRF: رمز عشوائي 32-byte محفوظ بالجلسة، يُطابَق بـ `secrets.compare_digest` (مقاوم لهجمات التوقيت).
- XSS: Jinja2 يقوم بـ auto-escape لكل المخرجات تلقائيًا + تنظيف إضافي للمدخلات النصية قبل التخزين.
- الجلسات: `HttpOnly` + `SameSite=Lax` (و`Secure` عند تفعيل HTTPS عبر `WGAME_HTTPS=1`).
